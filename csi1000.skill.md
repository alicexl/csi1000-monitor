中证1000 贴水策略监控 — 拉数据、算估值分位/期货贴水/期权增厚、生成信号和 Markdown 报告。

## Instructions

用户要跑中证1000贴水监控。用 `D:/workspace/csi1000-monitor/monitor.py` 完成任务。

**项目目录:** `D:/workspace/csi1000-monitor/`

### Step 1: 解析用户意图

根据 `$ARGUMENTS` 判断用户要干啥：

| 用户输入 | 执行命令 | 说明 |
|---|---|---|
| `scan` / 拉数据 / 更新数据 | `python monitor.py scan` | 拉估值+期货+期权入库 |
| `report` / 报告 / 看报告 | `python monitor.py report` | 生成 Markdown 报告 |
| `status` / 快查 / 看看 | `python monitor.py status` | 一行快速查 |
| `run` / 跑一下 / 全套 | `python monitor.py run` | scan + report 一键 |
| 空 / 不确定 | `python monitor.py run` | 默认全套 |

### Step 2: 执行

```bash
cd D:/workspace/csi1000-monitor && python monitor.py run 2>&1 | grep -v "UserWarning\|from pandas"
```

注意：akshare 网络调用可能需要 30-60 秒。周末/假日 `get_futures_daily` 返回空合约列表是正常的。

### Step 3: 展示结果

报告生成后，读取 `D:/workspace/csi1000-monitor/reports/csi1000_YYYY-MM-DD.md`，向用户展示关键数据：

1. **信号**：entry/warn_entry/wait（空仓）或 reduce/warn_reduce/switch/hold（持仓）
2. **估值面板**：PE_TTM / PE静态 / PB 的多区间分位
3. **期货合约**：IM 当月/下月/当季/下季的基差和年化贴水率
4. **卖 Call 增厚**：10% OTM call 的权利金、IV、年化增厚率、行权概率
5. **操作建议**：根据信号给出具体建议

### 策略逻辑（供解读数据用）

- **入场条件**：PE_TTM 10年分位 <50% 且 当月年化贴水 >5%
- **减仓条件**：PE_TTM 10年分位 >85%
- **合约切换**：当月剩余天数 <7 天时切下月
- **卖 Call 增厚**：持有 IM 多头时卖 10% OTM 当月 call 增厚收益

### config.yaml

用户持仓状态和阈值在 `D:/workspace/csi1000-monitor/config.yaml` 中维护：
- `position.status`: empty（空仓）| holding（持仓）
- `thresholds`: 入场/减仓/切换阈值
- 报告内容会根据持仓状态自动调整（空仓看入场条件，持仓看减仓/切换）

### 故障排查

- **scan 返回 0 合约**：周末/假日正常，工作日重试
- **option 部分缺失**：周末 akshare 期权接口可能无实时数据，工作日重试
- **PE/PB 数据为空**：检查网络连接，重试 `python monitor.py scan`
- **KeyError 列名**：akshare 可能改了列名，检查 `data_fetcher.py` 的 rename 映射
